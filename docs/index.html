3단계 외출 루틴

거창한 추적 서비스보다 먼저, 실제로 매일 쓸 수 있는 짧고 확실한 확인 흐름에 집중했습니다.

1

자주 챙기는 물건 선택

온보딩에서 기본 물품을 고르고, 물건별 힌트와 중요도를 설정합니다.

2

외출 직전 체크 세션

오늘 홈에서 챙김 모드를 시작해 미확인 물건을 빠르게 처리합니다.

3

이동 감지 후 한 번 더

사용자가 시작한 세션 중에만 위치 서비스를 실행하고 조건 충족 시 한 번만 알려줍니다.

Android 포트폴리오 포인트

Compose 화면 구현뿐 아니라 로컬 저장, foreground service, 권한 없는 수동 흐름, 테스트까지 MVP의 실제 제약을 담았습니다.

Kotlin + Compose

Material 3 기반의 온보딩, 오늘 홈, 챙김 세션, 내 물건, 설정 화면을 Signal Buddy 디자인 토큰으로 구성합니다.

Room물품과 챙김 세션 로컬 저장
DataStore온보딩 완료와 위치 감지 설정
StateFlowViewModel 중심 단방향 상태 흐름
Location Service세션 중에만 foreground 위치 확인
Notifications세션당 1회 리마인드
Tests도메인 정책, ViewModel, Compose UI 검증
Privacy by Design

위치 정보는 저장하지 않습니다.

  • 서버 없이 로컬 우선으로 동작합니다.
  • 위도·경도를 Room이나 로그에 저장하지 않습니다.
  • 시작 위치와 최신 판단 값은 서비스 메모리에서만 사용합니다.
  • 위치 추적은 사용자가 직접 챙김 세션을 시작한 뒤에만 실행됩니다.
  • 위치·알림 권한이 없어도 수동 체크리스트는 사용할 수 있습니다.
